  1. Que es un sistema de control de versiones?

    es un software que permite gestionar cambios en el código fuente y otros archivos a lo largo del tiempo.

  2. Cual es la diferencia entre git y github?

    Git es un sistema de control de versiones distribuido que permite rastrear cambios en archivos a lo largo del tiempo, realizar ramificaciones, fusionar cambios y más. GitHub, por otro lado, es una plataforma en línea que utiliza Git como su sistema de control de versiones subyacente, ofreciendo características adicionales como seguimiento de problemas, solicitudes de extracción y acciones para facilitar la colaboración en proyectos de desarrollo de software. En resumen, Git es la herramienta de línea de comandos para gestionar versiones, mientras que GitHub proporciona un entorno en línea para alojar y colaborar en proyectos utilizando Git.

  3. Cómo creo una rama en git?

    Mediante comandos git branch nombre_rama

  4. Cómo clono un repositorio?

    Mediante el comando git clone

  5. Cómo puedo bajar los ultimos cambios de un repositorio?

    Mediante el codigo git pull

  6. Cómo puedo gener un tag?

    Mediante git tag nombre_etiqueta

  7. Cómo hago una mezcla de ramas?

    Mediante el comando git merge

  8. Cómo puedo moverme a un commit especifico?

    Mediante git checkout

  9. Cómo puedo borrar una rama?

    Mediante branch -d

  10. Cómo puedo borrar un tag?

    mediante tag -d

  11. Cómo puedo ver los cambios que estan por enviarse al repositorio?

    Mediante git diff -cached

  12. Para que sirve el comando git remote -v?

    El comando git remote -v muestra la lista de repositorios remotos configurados en tu repositorio local de Git, junto con las URL asociadas a cada uno.

  13. Cómo le digo a git cual es mi email y correo?

    git config --global user.email "tu_correo@example.com"

  14. Cuales son los pasos para conectar git con github?

    Para conectar Git con GitHub, sigue estos pasos: Crea una cuenta en GitHub si aún no la tienes. Instala Git en tu sistema si aún no lo has hecho. Configura tu nombre de usuario y dirección de correo electrónico en Git usando git config. Opcionalmente, genera un par de claves SSH y agrégalas a tu cuenta de GitHub para una autenticación segura. Crea un repositorio en GitHub si no tienes uno. Clona el repositorio remoto en tu máquina local usando git clone. Realiza cambios en tus archivos locales, agrega los cambios con git add y realiza commits con git commit. Envía los cambios a GitHub usando git push.

  15. Cómo puedo crear un alias en git?

    git config alias.nombre_del_alias comando
